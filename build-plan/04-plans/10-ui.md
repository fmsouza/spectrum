# @launchkit/ui Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the React **atomic design system** — pure, presentational components from `atoms` → `templates` — that the desktop pages compose. Every component is dumb: typed props in, JSX out; events leave via callback props. No data fetching, no IPC, no global state (per `01-conventions/atomic-design.md`: **data enters only at the page level**, which lives in `apps/desktop`, not here).

**Architecture:** Strict atomic layering (`src/atoms`, `src/molecules`, `src/organisms`, `src/templates`), one component per file (`ComponentName.tsx`) co-located with `ComponentName.test.tsx`, re-exported through a per-level barrel (`atoms/index.ts`, …) and the package barrel `src/index.ts`. Every component has an explicit `Props` type. Components never carry secret values: provider props use a **secret-free display shape** (`Pick`ed from `Provider` — `id`/`name`/`sdkProvider` only, never `secrets`), matching the rule that the page passes display fields only. Long lists take a virtualization-ready prop so the page owns windowing (`01-conventions/performance.md`).

**Tech Stack:** TypeScript (strict, `jsx: react-jsx`), `react` + `react-dom` (pinned), `bun test` on happy-dom with `@testing-library/react` + `@testing-library/jest-dom` (registered by the root `test/setup.ts` preload from `phase0`).

> Depends on: `types`, `utils` (only — never `ipc`/`config`/`proxy`; see `02-monorepo/boundaries.md` rule 5). Read `01-conventions/atomic-design.md`, `typescript.md`, `tdd.md`, `performance.md`, and the `Provider`/`ModelAlias`/`HarnessDefinition`/`Session` shapes in `04-plans/01-types.md`.
> Create the package via `launchkit-new-package`: `packages/ui`, deps `react` + `react-dom` (pinned), `@launchkit/types` + `@launchkit/utils` as `workspace:*`. Its `tsconfig.json` extends the React preset (`@launchkit/tsconfig/react.json`: `jsx: react-jsx`, DOM libs) and `references` `types` + `utils`.
> RED for every task is `bun test packages/ui`. All component tests import `{ describe, it, expect, mock }` from `"bun:test"` and `{ render, screen, fireEvent }` from `"@testing-library/react"`; `toBeInTheDocument`/`toHaveValue`/`toBeDisabled` matchers come from the preload. Tests are named `it("does X when Y happens")`. `afterEach(cleanup)` is global (preload) — no per-file teardown.

import { useState } from "react"
import type { ApiFormat } from "@launchkit/types"
import { Button } from "../atoms/Button"
import { Select } from "../atoms/Select"
import { TextInput } from "../atoms/TextInput"
import { FormField } from "../molecules/FormField"

export type HarnessFormValues = {
  readonly name: string
  readonly command: string
  readonly apiFormat: ApiFormat
  readonly defaultAlias: string
}

export type HarnessFormProps = {
  readonly initialValues: HarnessFormValues
  readonly onSubmit: (values: HarnessFormValues) => void
  readonly onCancel: () => void
}

const API_FORMAT_OPTIONS = [
  { value: "anthropic", label: "Anthropic" },
  { value: "openai", label: "OpenAI" },
] as const

export const HarnessForm = ({ initialValues, onSubmit, onCancel }: HarnessFormProps): JSX.Element => {
  const [values, setValues] = useState<HarnessFormValues>(initialValues)
  const update = <K extends keyof HarnessFormValues>(key: K, value: HarnessFormValues[K]): void =>
    setValues((prev) => ({ ...prev, [key]: value }))

  const submit = (): void => {
    if (values.name.trim() === "" || values.command.trim() === "") return
    onSubmit(values)
  }

  return (
    <form
      onSubmit={(e) => {
        e.preventDefault()
        submit()
      }}
    >
      <FormField id="harness-name" label="Name">
        <TextInput id="harness-name" value={values.name} onChange={(v) => update("name", v)} />
      </FormField>
      <FormField id="harness-command" label="Command">
        <TextInput id="harness-command" value={values.command} onChange={(v) => update("command", v)} />
      </FormField>
      <FormField id="harness-format" label="API format">
        <Select
          id="harness-format"
          value={values.apiFormat}
          options={API_FORMAT_OPTIONS}
          onChange={(v) => update("apiFormat", v as ApiFormat)}
        />
      </FormField>
      <FormField id="harness-alias" label="Default alias">
        <TextInput id="harness-alias" value={values.defaultAlias} onChange={(v) => update("defaultAlias", v)} />
      </FormField>
      <Button onClick={() => submit()}>Save</Button>
      <Button variant="secondary" onClick={() => onCancel()}>Cancel</Button>
    </form>
  )
}
```

> The `Button`s are `type="button"` (atom default), so clicking "Save" runs `submit()` directly rather than relying on native submit; the `<form onSubmit>` mirrors the same guard for Enter-key submission. `apiFormat`'s value is narrowed to `ApiFormat` at the `Select` boundary — the option list is the two `ApiFormat` members, so the cast is total.

import type { Session } from "@launchkit/types"
import { Badge } from "../atoms/Badge"
import { EmptyState } from "../molecules/EmptyState"

export type SessionTableProps = {
  readonly sessions: readonly Session[]
  /** Render at most this many rows; the page virtualizes longer histories (performance.md). */
  readonly maxVisible?: number
}

export const SessionTable = ({ sessions, maxVisible }: SessionTableProps): JSX.Element => {
  if (sessions.length === 0) {
    return <EmptyState title="No sessions yet" hint="Launched harnesses will appear here." />
  }
  const visible = maxVisible === undefined ? sessions : sessions.slice(0, maxVisible)
  const hidden = sessions.length - visible.length

  return (
    <div>
      <table>
        <thead>
          <tr>
            <th>Harness</th>
            <th>Alias</th>
            <th>Started</th>
            <th>Status</th>
          </tr>
        </thead>
        <tbody>
          {visible.map((session) => (
            <tr key={session.id}>
              <td>{session.harnessId}</td>
              <td>{session.alias}</td>
              <td>{session.startedAt}</td>
              <td>
                {session.endedAt === undefined ? (
                  <Badge tone="info">running</Badge>
                ) : (
                  <Badge tone={session.exitCode === 0 ? "success" : "danger"}>{`exit ${session.exitCode ?? "?"}`}</Badge>
                )}
              </td>
            </tr>
          ))}
        </tbody>
      </table>
      {hidden > 0 ? <p>{`+${hidden} more`}</p> : null}
    </div>
  )
}
```

# @launchkit/ui

**Responsibility:** React atomic design system — pure, presentational components (atoms → templates) that the desktop pages compose.

**Public API (barrel `src/index.ts`):** atoms (`Button`, `TextInput`, `Select`, `Badge`, `StatusDot`, `Spinner`, `Label`), molecules (`FormField`, `ProviderCard`, `AliasRow`, `EmptyState`), organisms (`ProviderList`, `AliasTable`, `HarnessForm`, `SessionTable`), templates (`AppShell`, `SettingsLayout`) — each re-exported via its level barrel and the package barrel.

**Effects owned:** none. No data fetching, no IPC, no global state.

**Local rules:** atoms→organisms are pure/presentational — typed props in, JSX out, events out via callback props; follow `01-conventions/atomic-design.md`. Data enters only at the page level (in `apps/desktop`), never here. Every component has an explicit `Props` type (no `any`). Provider props use a secret-free display shape (`Pick<Provider, "id" | "name" | "sdkProvider">`) — no `secrets` field ever reaches a component. One component per file with a co-located `*.test.tsx`. Long lists (`SessionTable`) take a `maxVisible` prop; the page virtualizes (`01-conventions/performance.md`).
